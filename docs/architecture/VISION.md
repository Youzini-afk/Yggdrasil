# 愿景

> [English](./VISION.en.md) · [中文](./VISION.md)

Yggdrasil 是一个面向 AI 原生世界、游戏、故事与游玩的可扩展创作平台。

平台中心稳定、克制，对内容没有立场。所有有意义的概念都生活在能力包里，内核以平等的方式承载它们。

## Yggdrasil 是什么

一个承载能力包的内核。

一份让客户端、能力包、外部系统平等接入的公开协议。

一份保留所发生之事的事件日志。

一个面向激进 AI 原生体验的创作底座——这些体验由谁创造、什么形态，平台不预设。

## Yggdrasil 不是什么

不是应用，不是聊天工具，不是 SillyTavern 替代品，也不是把题材内置进框架的产品。它的内核里没有特权官方内容。

平台不会给出范本体验。内核对角色、世界、提示词、模型、agent、记忆这些概念没有任何立场。这些都是能力包关心的事。

## 「激进的创作自由」在这里意味着什么

创作者不被局限于平台想象的形态。

创作者可以：

- 定义自己的题材、循环、规则和呈现方式；
- 把 AI 行为当作积木来组合；
- 审视、fork、改写、重组任何体验；
- 用自己的能力包替换或覆盖任何官方包；
- 发布新的能力、新的事件类型、新的扩展点；
- 在同一个会话里混用多个能力包，谁也不享有特权。

平台的职责是让这一切成为可能，而不是亲自交付体验。

## 为什么要内核 + 能力包

封闭的框架决定了媒介长什么样。Yggdrasil 拒绝这样做。

把所有含义都放进能力包——包括官方包——保持了媒介的开放，也让平台诚实：如果一个官方「对话运行时」可以被替换，或者能与第三方「世界模拟器」共存，那内核就不是暗中的掌控者。

这是在时间维度上守护创作自由的方式。

## 适用范围

Yggdrasil 设计为可以承担以下角色：

- 本地平台 host；
- 讲公开协议的无头服务；
- 嵌入更大产品中的库；
- 让外部系统作为能力包或客户端接入的开放协议端点。

四种角色共用同一份契约。

## 延后的能力家族

下面这些方向有价值，但属于能力包，不是内核关心的事。它们要等内核与能力包层稳定之后才会到来。

- YdlTavern——一个跑在 Yggdrasil 之上、兼容 SillyTavern 资源与扩展的独立接入项目，见 [`../tavern/TAVERN_COMPAT.md`](../tavern/TAVERN_COMPAT.md)；
- agent 集成包家族（pi 或其他）；
- 游戏引擎桥接包家族（UE5、Godot、Unity、Web 客户端）；
- 官方对话运行时包；
- 官方审查器与创作者 UI。

每一项都将作为普通能力包来构建和评判。没有一项会获得内核特权。

## 非目标

内核不会发布聊天体验、世界模拟器、导演、记忆模型、SillyTavern 兼容层、外部引擎桥接，也不会有官方 UI。

这些都适合以能力包的形态存在；都不适合长在内核里。

## 对当前代码的态度

Rust 工作区目前处于 Platform Foundation Alpha：仅内核的事件/会话、基于清单的能力包、真实的 `rust_inproc` 与子进程执行、钩子机制、SQLite 事件日志、有权限的身份模型、surface 贡献、提案/审批生命周期、资产/分支/projection 底座，以及一份走公开协议的 Web shell。当前的纪律是防止契约漂移——surface、提案、分支、资产、projection 必须保持通用形态，内容形态的语义不能渗进内核，官方包只能用第三方包也能用的东西。

## 成功的样子

当一个创作者能造出平台作者从未预见的东西，把它作为能力包发布，与官方包并行运行而不被当二等公民对待，Yggdrasil 就成功了。
